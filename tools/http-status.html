📦 HTTP Status Codes / HTTP 状态码

💡 开发工具 · L1 查表
🔍
输入数字或关键词实时过滤状态码
1xx Informational / 信息响应 2
100
Continue
继续。客户端应继续请求
提示：发送大请求体前先发送 Expect: 100-continue 头
📋
101
Switching Protocols
切换协议。服务器将遵从 Upgrade 头
提示：WebSocket 连接时使用，检查 Upgrade 头是否正确
📋
2xx Success / 成功 3
200
OK
请求成功
提示：标准成功响应，无需额外操作
📋
201
Created
资源创建成功
提示：POST/PUT 创建资源后返回，Location 头指向新资源
📋
204
No Content
请求成功但无返回内容
提示：DELETE 成功后常用，响应体为空
📋
3xx Redirection / 重定向 3
301
Moved Permanently
永久重定向
提示：SEO 使用 301 传递权重，确保 Location 头正确
📋
302
Found
临时重定向
提示：临时跳转，后续请求仍使用原 URL
📋
304
Not Modified
资源未修改，使用缓存
提示：验证 If-None-Match/If-Modified-Since 缓存头
📋
4xx Client Error / 客户端错误 11
400
Bad Request
请求语法错误
修复：检查请求参数格式、JSON 语法、必填字段
📋
401
Unauthorized
未授权，需身份验证
修复：检查 Token 是否有效、过期，确认 Authorization 头
📋
403
Forbidden
禁止访问，权限不足
修复：检查用户角色权限、IP 白名单、CORS 配置
📋
404
Not Found
资源不存在
修复：检查 URL 路径是否正确、资源是否被删除或移动
📋
405
Method Not Allowed
请求方法不允许
修复：确认 API 支持的 HTTP 方法，检查路由配置
📋
408
Request Timeout
请求超时
修复：检查网络连接、增加超时时间、优化服务端处理速度
📋
409
Conflict
资源冲突
修复：检查并发操作、唯一约束冲突、版本控制
📋
410
Gone
资源已永久删除
提示：资源不再可用，更新客户端引用或文档
📋
422
Unprocessable Entity
无法处理的实体
修复：检查数据验证规则、字段类型、业务逻辑约束
📋
429
Too Many Requests
请求过于频繁
修复：降低请求频率、实现指数退避、检查 Rate Limit 头
📋
451
Unavailable For Legal Reasons
因法律原因不可用
提示：因法律要求屏蔽资源，需合规审查
📋
5xx Server Error / 服务端错误 4
500
Internal Server Error
服务器内部错误
修复：查看服务器日志、检查代码异常、数据库连接
📋
502
Bad Gateway
网关错误
修复：检查上游服务状态、网关/代理配置、负载均衡
📋
503
Service Unavailable
服务暂时不可用
修复：服务可能过载或维护，检查 Retry-After 头
📋
504
Gateway Timeout
网关超时
修复：上游服务响应过慢，检查超时配置和服务性能
