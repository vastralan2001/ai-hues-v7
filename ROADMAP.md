## Phase 2 规划 (Week 3-4)

### P0 — 用户体验完善 (Week 3)

| # | 功能 | 说明 | 技术方案 |
|---|------|------|----------|
| 1 | **i18n全局生效** | 目前只有首页和i18n.js完成了双语切换，需要让40+个工具/游戏页面都引用i18n.js并添加data-zh属性 | 批量给tools/*.html和games/*.html添加 `<script src="../js/i18n.js"></script>` 和 data-zh |
| 2 | **Login状态全局生效** | 登录后所有页面右上角显示用户名，未登录时显示"Login" | 每个页面添加 `<script src="../js/auth.js"></script>`，JS检测localStorage中的current_user |
| 3 | **导航栏统一** | 目前新工具页面和旧工具页面导航不一致，需要统一为新版导航（含Wishlist/Ranking/Login） | 重写所有旧工具页面的nav部分 |
| 4 | **收藏功能** | 用户可以收藏常用工具，在首页或个人中心快速访问 | localStorage存收藏列表 |
| 5 | **使用统计埋点** | 记录每个工具的使用次数、时长，为排行榜提供真实数据 | localStorage记录，每次打开工具+1 |

### 不做清单

| 功能 | 原因 |
|------|------|
| 后端API/数据库 | 纯静态站点，所有数据用localStorage |
| 用户认证后端 | 纯前端模拟，不存储真实密码 |
| 付费系统 | 目前Credit纯免费发放 |
| 实时多人游戏 | 超出静态站范围 |
| 图片压缩/OCR/PDF | 需要后端处理 |
