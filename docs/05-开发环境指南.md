## 快速开始（5 分钟）

### 1. Clone 项目

```bash
git clone https://github.com/vastralan2001/ai-hues-v7.git
# 或者 SSH: git@github.com:vastralan2001/ai-hues-v7.git
cd ai-hues-v7
```

### 2. 安装依赖

```bash
npm install
# 或 pnpm install（更快）
```

### 3. 配置环境变量

```bash
cp .env.example .env.local
```

编辑 `.env.local`：

```bash
# 数据库（开发时可以用本地 PostgreSQL 或 Vercel Postgres）
DATABASE_URL="postgresql://user:password@localhost:5432/aihues"

# NextAuth 密钥（生成命令见下方）
NEXTAUTH_SECRET="your-random-secret"
NEXTAUTH_URL="http://localhost:3000"

# OAuth（可选，开发时可以先不配置）
GITHUB_CLIENT_ID=""
GITHUB_CLIENT_SECRET=""
```

**生成 NEXTAUTH_SECRET**:
```bash
openssl rand -base64 32
```

### 4. 初始化数据库

```bash
# 创建数据库
npx prisma migrate dev --name init

# 插入种子数据（57 个工具 + 10 个成就）
npx prisma db seed

# 打开数据库 GUI（可选）
npx prisma studio
```

### 5. 启动开发服务器

```bash
npm run dev
```

访问 http://localhost:3000

## Git 工作流

```bash
# 1. 基于 main 创建功能分支
git checkout main
git pull origin main
git checkout -b feat/add-search-filter

# 2. 开发 + 提交
git add .
git commit -m "feat: 添加发现页筛选功能"

# 3. 推送并创建 PR
git push origin feat/add-search-filter
# 然后在 GitHub 上创建 Pull Request

# 4. 合并后清理

git pull origin main
```

### Commit Message 规范

```

docs:     文档更新

refactor: 重构（非 feat/fix）
test:     测试相关
chore:    构建/工具/依赖更新
```

## 开发规范

### TypeScript

- 所有新代码用 TypeScript
- 避免 `any`，用 `unknown` + 类型守卫
- 接口命名用 PascalCase（`UserProfile`）
- 函数返回类型明确标注

### 组件

- 优先用 Server Components（默认）
- 需要交互的用 `'use client'` Client Components
- 组件文件用 PascalCase（`ToolCard.tsx`）
- props 用 interface 定义

### CSS

- 用 Tailwind CSS 工具类
- 复杂样式用 `cn()` 工具函数（tailwind-merge + clsx）
- 主题变量在 globals.css 的 `:root` 和 `[data-theme]` 中定义
- 避免内联 `style={}`

### Server Actions

- 所有数据操作放在 `app/actions/` 目录
- 文件名按功能命名（`tools.ts`, `ratings.ts`）
- 函数名用 camelCase（`submitRating`）
- 返回统一格式：`{ success: true, data }` 或 `{ success: false, error }`
- 敏感操作必须 `const session = await auth()` 验证登录

## 常见问题

### Q: Prisma Client 找不到？

```bash
npx prisma generate
```

### Q: 数据库连接失败？

检查 `.env.local` 的 `DATABASE_URL` 格式：
```
postgresql://USER:PASSWORD@HOST:PORT/DATABASE?schema=public
```

本地开发可以用 Docker 启动 PostgreSQL：
```bash
docker run -d \
  --name aihues-db \
  -e POSTGRES_USER=aihues \
  -e POSTGRES_PASSWORD=aihues123 \
  -e POSTGRES_DB=aihues \
  -p 5432:5432 \
  postgres:15
```

### Q: OAuth 登录不工作？

开发时可以先跳过 OAuth：
1. 不配置 `GITHUB_CLIENT_ID` 和 `GOOGLE_CLIENT_ID`
2. NextAuth 会自动回退到 Credentials Provider（邮箱+密码）
3. 或者用 mock 用户登录（开发模式专用）

### Q: 如何只测试某个工具？

```bash
# 直接访问工具页面
http://localhost:3000/tools/json-formatter

# 或运行单个测试
npx vitest run --reporter verbose json-formatter
```
